# Historical Planning Archive

This directory contains historical implementation plans and design notes from earlier development phases.

These files are preserved for context only. They are not the active backlog, and unchecked checklist items inside older plan files do not mean work is still pending. Current project status should be read from the root `README.md`, open GitHub issues, and the current `main` branch.

Use these documents when you need to understand why a feature was built a certain way or recover implementation context from a past phase. Do not resume one of these plans as active work unless a new issue or branch explicitly reactivates it.

## Contents

- `specs/` - archived design notes and implementation direction for completed or superseded phases.
- `plans/` - archived execution plans. Checklist state is intentionally preserved as written at the time and should not be used as current task status.
